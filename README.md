# injector

A Windows DLL (`dbghelp.dll`) that proxies the system `dbghelp.dll` and patches the Evolve process at load time so it accepts pak files signed with the revival RSA-1024 keypair instead of the vanilla key.

## How it works

Evolve loads `dbghelp.dll` from its own `bin64_SteamRetail/` directory before the system path. This DLL:

1. Proxies every export to the real system `dbghelp.dll`, so all debugger and crash-reporting functionality works normally
2. At attach time, scans the game's loaded image for the vanilla RSA-1024 public key (140-byte DER sequence, known constant)
3. Loads `revival.pub` from the same directory
4. Overwrites the vanilla key in memory with the revival public key

After this patch, the game's pak verification accepts files re-signed by `evolve-pak rekey`.

## Build

Requires MinGW-w64:

```bash
make
```

Output: `dbghelp.dll`. Place it in `bin64_SteamRetail/` alongside `revival.pub` (generated by `evolve-pak keygen`).
